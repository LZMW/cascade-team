# Cascade (级联战队) - 6A框架专家团队

## 概述

**Cascade（级联战队）** 是一个按照**统一项目开发工作流（6A框架）**行事的专家团队。团队由5位专业领域专家组成，覆盖软件项目全生命周期。

### 6A框架

```
┌─────────────────────────────────────────────────────────────────┐
│                     6A 项目开发工作流                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Align    (对齐阶段)     模糊需求 → 精确规范    → Anchor     │
│         ↓                                                       │
│  2. Architect (架构阶段)    共识文档 → 系统架构    → Atlas      │
│         ↓                                                       │
│  3. Atomize  (原子化阶段)   架构设计 → 任务拆解    → Prism      │
│         ↓                                                       │
│  4. Approve  (审批阶段)     原子任务 → 人工审查    → 用户审批   │
│         ↓                                                       │
│  5. Automate (自动化执行)   按节点执行 → 代码实现   → Forge      │
│         ↓                                                       │
│  6. Assess   (评估阶段)     执行结果 → 质量评估    → Scale      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

|------|------|------|----------|------------|

## 协作模式

### 模式 A - 全流程链式协作（推荐）

适用于完整项目开发，按6A框架顺序执行：

```
用户请求 → 协调器分析
  → Anchor (对齐) → Atlas (架构) → Prism (拆解)
  → [用户审批]
  → Forge (执行) → Scale (评估)
  → 汇总返回
```

### 模式 B - 单阶段专家

适用于用户只需要某个特定阶段的工作：

```
用户请求 → 协调器分析 → 触发单个专家 → 返回结果
```

### 模式 C - 阶段跳跃

适用于已有部分成果，从中间阶段开始：

```
用户请求（已有设计文档）→ 协调器分析
  → Prism (基于已有设计拆解)
  → [用户审批]
  → Forge → Scale
```

## 质量门控

每个阶段完成后都有明确的质量门控：

| 阶段 | 质量门控 |
|------|----------|
| Align | 需求边界清晰、验收标准可测试、关键假设已确认 |
| Architect | 架构图清晰准确、接口定义完整、与现有系统无冲突 |
| Atomize | 任务覆盖完整需求、依赖关系无循环、每个任务可独立验证 |
| Approve | 执行检查清单通过、最终确认清单完成 |
| Automate | 代码编译通过、测试全部通过、文档同步更新 |
| Assess | 所有需求已实现、验收标准满足、最终交付物完整 |

## 使用示例

### 启动完整项目

```
用户: 我想开发一个用户认证系统
协调器: 我将启动6A全流程协作模式...
  1. 使用 cascade-anchor 进行需求对齐
  2. 使用 cascade-atlas 设计系统架构
  3. 使用 cascade-prism 拆解任务
  4. [等待您的审批确认]
  5. 使用 cascade-forge 实现代码
  6. 使用 cascade-scale 评估质量
```

### 单阶段调用

```
用户: 帮我对齐这个项目的需求边界
协调器: 使用 cascade-anchor 专家进行需求对齐...
```

## 核心原则

1. **绝不允许项目延期** - 严格按规划执行，识别风险及时调整
2. **绝不允许超出计划** - 开发范围严格匹配规划
3. **绝不允许出错** - 多轮自检，遇错立即暂停修复
4. **文档同步** - 代码变更同时更新所有相关文档
5. **测试优先** - 先写测试，后写实现
